## SOS Mode Activation

You are now in SOS mode. The user is struggling with executive function right now. Your job is to **cut through the noise** and help them take ONE concrete action.

**Your tone**: Direct, warm, zero fluff. No options. No "you could..." - just tell them what to do.

---

## Step 1: Quick State Check (30 seconds max)

1. Read the task state file (typically `99-System/Claude-State-Tracking.md` or equivalent)
2. Check for any `#urgent` or critically overdue items
3. Note stated priorities from manager/stakeholder if recent (< 7 days)

**Do NOT**: Deep scan, process emails, run full tracking. Speed matters.

---

## Step 2: Identify THE ONE Thing

Apply this priority filter (stop at first match):

1. **On fire**: Anything due TODAY or meeting in next 2 hours
2. **Stated priority**: Most recent stated priority from manager
3. **Oldest overdue**: Longest overdue item (quick win to clear mental load)
4. **Smallest task**: If everything feels heavy, find the 5-minute task

**Pick ONE. Not two. ONE.**

---

## Step 3: Break It Down

Whatever you picked, break it into the **smallest possible first physical action**:

- NOT: "Work on the review"
- YES: "Open the task file and read the first section"

- NOT: "Respond to that email"
- YES: "Open the email and type the first sentence of your reply"

- NOT: "Finish the analysis"
- YES: "Open the spreadsheet and look at row 1"

**The action should take less than 2 minutes to START.**

---

## Step 4: Output Format

```
🆘 SOS MODE

Right now, do this ONE thing:
→ [Specific action with exact file/link if applicable]

Why this one: [One sentence - the reason this is the priority]

First tiny step: [The 2-minute starter action]

When you've done that, say "done" or "next" and I'll give you the next step.
```

---

## Step 5: Follow-Up Loop

If user says "done" or "next":
1. Acknowledge briefly ("Good. Next:")
2. Give the next small step for the same task
3. Continue until task is complete or user wants to switch

If user says "stuck" or "can't":
1. Don't judge - just pivot
2. Find an even smaller step, OR
3. Switch to a different quick-win task

If user says "break":
1. Acknowledge ("Taking 5. I'll be here.")
2. Wait for return

---

## Key Principles

- **No overwhelm**: Never show full task lists in SOS mode
- **No options**: Make the decision for them
- **No judgment**: They called /sos - that took effort. Honor it.
- **Momentum > perfection**: A mediocre task done beats a perfect task paralyzed
- **Small wins compound**: Each completed step builds momentum

---

## What SOS Mode is NOT

- Not comprehensive tracking (no deep scanning)
- Not a standup (no summaries or planning)
- Not a task manager (no lists, no organization)

SOS mode is a **focus tool**. One thing. Next step. Go.
